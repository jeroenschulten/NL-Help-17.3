Security Group Properties dialog - General tab

You use the General tab of the Security Group Properties dialog to edit the name of the selected security group and to specify the access rights that members of the security group have over certain aspects of the project.

You can give the members of a security group particular rights for bars and tasks with a particular code from a specified "security library", in addition to the access rights that you define using the Chart Access Rights tab.

Explanation of the fields on the dialog
Field Description
Name Enter the name of the security group.
Security library

To assign access rights for a particular code or code library (in addition to any chart access rights), select the appropriate code library in this field. You can then click Properties to display the Code Library Rights tab, in which you can specify the access rights for a specific code or for the whole library.

 

For example, you might have a code library called People containing a code for each employee in the company, so you could give the members of each security group full access to any bars, tasks or allocations that have their code. These rights will be in addition to any access rights set up per chart. So a member of a security group might have only read access to a chart, but full access to any bars, tasks or allocations with their code within the chart.

 

If you select a code library in this field, you must use the Library Access Rights tab to make the library read-only, to prevent the members of the security group from changing their access rights for the library.

Publish

Select this check box to allow members of this security group to publish details of permanent resource allocations to Microsoft Outlook® and via SMTP email, or clear the check box if you do not want them to be able to do this, or if you do not use this feature at all.

 

 

Specifying whether users can publish details of permanent resource allocations to Microsoft Outlook® or via SMTP email

Archive/restore

Select this check box to enable members of this security group to archive and restore data using the application, or clear the check box to prevent them from doing so.

 

Users can only archive data from charts and summary groups to which their security group has Read rights. This is defined on the Chart Access Rights tab of the dialog. You can only archive and restore data if you use Enterprise.

 

Specifying whether users can archive and restore data

Task Pool Access

Click the appropriate radio button to specify the access rights that members of this security group should have over task pools.

 

Specifying whether users can create, view and modify task pools

Save project macros

Select this check box to allow members of this security group to save (and therefore overwrite) project macros, or clear the check box if you do not want them to be able to overwrite project macros. Preventing users from saving project macros may be useful if you have company-wide macros that you do not want users to overwrite accidentally.

 

 

Specifying whether users can create project macros

Create project level views

Select this check box to specify that members of this security group are able to create project views, or clear the check box to specify that they can only create user views.

 

Specifying whether users can create project views

Embed/delete border files

Select this check box to specify that members of this security group are able to embed border files into the project (and delete embedded border files from the project), or clear the check box to specify that they cannot embed and delete border files.

Edit project date controls

Select this check box to specify that members of this security group are able to edit the fields in the 'Dates' section of the Properties tab of the Backstage view, the corresponding fields on the Properties tab of the Properties dialog and - when viewing the properties of the programme chart - the date-related fields on the Project tab of the Chart/Project Properties dialog, or clear the check box if you do not want them to be able to do this.

 

Editing the properties of projects and charts

Edit timeline

Select this check box to specify that members of this security group are able to add tasks to a project timeline, remove tasks from a timeline and format a timeline, or clear the check box if you do not want them to be able to do this.

 

If you clear this check box, members of this security group will still be able to view the project timeline, export the timeline, 'go to' a task in the timeline and view a task's properties in the timeline.

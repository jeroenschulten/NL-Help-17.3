Bepalen welke publicatie methode te gebruiken

Om te bepalen welke publicatie methode voor details van resources toewijzingen het beste bij u past, moet u de software die op uw computers en servers draait eerst analyseren:

Het analyseren van de software die wordt gebruikt voor uw e-mail en domein servers

Als uw e-mail servers Microsoft Exchange Versie 5.5 of eerdere versies gebruiken, of als uw domein servers Windows® NT4 of eerdere versies gebruiken, kunt u alleen toewijzingen publiceren via SMTP e-mail. Dit komt doordat deze versies niet langer ondersteund worden door Microsoft.

Als u een andere methode dan SMTP e-mail wilt gebruiken voor publicatie, moet u de software die uw e-mail of domein server gebruikt upgraden. Kijk op de website van Microsoft voor details over welke server operatie systemen en e-mail servers wel ondersteund worden door Microsoft en voor informatie over hoe u uw huidige systeem naar de nieuwe domein server en e-mail server kunt migreren.

De software op de computer van de project manager analyseren

In dit geval verwijst de term "project manager" naar de persoon die gaat gebruiken om de toewijzingen naar de resources te publiceren.

Als de project manager een computer heeft die draait op Windows NT4, of als deze gebruik maakt van Microsoft Outlook 97, kunt u alleen toewijzingen via SMTP e-mail versturen.

Als de computer van de project manager andere e-mail software gebruikt dan Microsoft Outlook, kunt u alleen toewijzingen via SMTP e-mail publiceren. De project manager kan dan ook geen resources importeren uit hun adresboek. Let erop dat deze beperkingen ook van toepassing zijn wanneer de computer van de project manager draait op Outlook Express®.

Let erop dat er problemen kunnen ontstaan wanneer meer dan één versie van Microsoft Outlook tegelijkertijd op de computer van de project manager geïnstalleerd is.

Het analyseren van de software gebruikt op de computer van de resources

In dit geval verwijst de term "resource" naar de stafmedewerkers van het bedrijf die met de projecten zullen gaan werken en die weergegeven worden binnen een project als aparte resource records. Dit zijn de mensen naar wie de project manager informatie zal publiceren.

Als de computer van één van de resources andere e-mail software gebruikt dan Microsoft Outlook, bijvoorbeeld Lotus Notes®, Thunderbird of Microsoft Outlook Express, dient u zodanig in te stellen dat alle toewijzingen gepubliceerd worden via SMTP e-mail.

Als één van de computers van uw resources Windows 98 of Microsoft Outlook 97 gebruikt, dient u zich bewust te zijn van het volgende:

  • Wilt u dat uw resources de mogelijkheid hebben om de voortgang te rapporteren over taken die aan hen zijn toegewezen? Zo ja, dan dient u zodanig in te stellen dat alle toewijzingen als taken gepubliceerd worden in de Microsoft Outlook taken lijst van de resource. Wanneer resources Windows 98 of Microsoft Outlook 97 gebruiken, wordt met klem geadviseerd om zodanig in te stellen dat er naar een gedeelde of publieke map op de Exchange server gepubliceerd wordt, in plaats van publicatie naar de Microsoft Outlook map van iedere individuele gebruiker.
  • Wilt u dat uw resources de tijden van de toewijzing weergegeven krijgen in hun eigen kalender? Zo ja, dan dient u zodanig in te stellen dat toewijzingen als afspraken in de Microsoft Outlook kalender van de resource weergegeven worden. Wanneer resources gebruik maken van Windows 98 of Microsoft Outlook 97, wordt met klem geadviseerd om zodanig in te stellen dat er naar een gedeelde of publieke map op de Exchange server gepubliceerd wordt, in plaats van publicatie naar de Microsoft Outlook map van iedere gebruiker.
  • Wilt u dat uw resources e-mails ontvangen waarin ze geïnformeerd worden over de aan hen toegewezen taken? Zo ja, dan dient u zodanig in te stellen dat toewijzingen door SMTP e-mail gepubliceerd worden.

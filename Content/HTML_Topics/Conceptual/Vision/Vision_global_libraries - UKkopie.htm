Managing global libraries with

If you use to manage multiple projects, you can use to manage global libraries. This can save you a considerable amount of time.

How global libraries work

If you do not use global libraries, each individual project has its own calendar, code library, progress period, quality check, permanent resource and consumable resource libraries, stored in the project file. You maintain these libraries by opening individual projects in and using Library Explorer. This way of working may suit you if you manage a disparate range of projects, with each project using different calendars, codes, resources, etc.

If your projects all use the same set of calendars, codes, resources, etc, using global libraries can save you a significant amount of time, as this allows you to maintain a single - 'global' - library that contains the calendars, codes, resources, etc that are used throughout your projects. If you need to add a new permanent resource, edit a calendar or rename a code, using global libraries means that you only need to do this once, rather than having to make the same change in each individual project.

If you enable the use of global libraries in , creates a global library file that contains the libraries you choose to include.

When you check out or download an programme in , the global library file is checked out automatically alongside the programme and its contents are merged into the programme. You can view and edit global library objects - for example, calendars, codes and resources - in using the Global Library Explorer. When you subsequently check in an programme, the global library file is updated with any changes you have made to the global libraries while editing the programme. Your changes are then applied automatically to all other programmes when they are checked out.

 Configuring to use global libraries

You use to enable the use of global libraries, to specify which library object types are included in the global library file, and to specify which users are able to manage the global library file. Refer to the Help for information on how to do this.

 Editing a global library file

  1. Click the File tab, then click Open.
  2. Click Vision. A button appears for you to log into .
  3. Log into . A list of programmes to which you have access appears. If you are already logged into , the programme list appears without you having to log in.
  4. Click a programme to check it out. The programme is checked out, along with the global library file, the contents of which are merged into the programme.
  5. Once the programme has been checked out and appears in , return to the Open tab of the Backstage view and click .

If this is the first time you have edited the global library file, you will need to populate it with library objects. You can do this by creating them from scratch; or by opening one or more other projects in and copying and pasting library objects from Library Explorer into the Global Library Explorer; or by a combination of both these things.
